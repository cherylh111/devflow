# Repository Package Map

These guidelines describe the current DevFlow repository shape and how package
keys in `.devflow/config.yaml` map to source code and specs.

## Purpose

Keep DevFlow package/spec routing aligned with the actual repository so
`get_context.py --mode packages`, task package binding, and future AI sessions
load the right guidance.

## Package Map

| Package key | Path | Package name | Spec path | Purpose |
| --- | --- | --- | --- | --- |
| `cli` | `packages/cli` | `@enpd/devflow` | `.devflow/spec/cli/**` | User-facing CLI, templates, migrations, release scripts, and platform configurators. |
| `core` | `packages/core` | `@enpd/devflow-core` | `.devflow/spec/core/**` | Reusable SDK/domain primitives for channel, mem, task, and testing APIs. |
| `docs-site` | `docs-site` | `devflow-docs` | `.devflow/spec/docs-site/**` | Mintlify documentation site. This directory is a git submodule, not part of `pnpm-workspace.yaml`. |

## Workspace And Submodules

- `pnpm-workspace.yaml` declares `packages/*`, so the pnpm workspace packages
  are `packages/cli` and `packages/core`.
- `.gitmodules` declares `docs-site` and `marketplace` as submodules.
- `docs-site` has its own package manifest and DevFlow specs, so it is a
  configured DevFlow package with `type: submodule`.
- `marketplace` is a submodule but currently has no package manifest and no
  DevFlow package-scoped specs. Do not add it to `.devflow/config.yaml` unless
  it gains package-level implementation ownership or specs.

## Contracts

- `.devflow/config.yaml` uses stable DevFlow package keys (`cli`, `core`,
  `docs-site`) rather than npm package names. These keys are part of spec paths
  and task package bindings.
- `default_package` is `cli`; new tasks without `--package` bind to CLI because
  most DevFlow workflow/runtime changes are CLI-owned.
- Existing tasks keep their frozen `task.json.package`. Changing
  `.devflow/config.yaml` does not retroactively rebind old tasks.
- Package-context output is generated by
  `.devflow/scripts/common/packages_context.py`, which scans
  `.devflow/spec/<package>/<layer>/index.md` for each configured package.

## Checks

```bash
python3 ./.devflow/scripts/get_context.py --mode packages
python3 "${CODEX_HOME:-$HOME/.codex}/skills/devflow-spec-maintainer/scripts/scan_spec_drift.py"
git diff --check -- .devflow/config.yaml .devflow/spec
```

If a future checkout provides a package-map checker under the DevFlow scripts
directory, run it as part of package-map validation. That helper is not
currently present in this repo.
