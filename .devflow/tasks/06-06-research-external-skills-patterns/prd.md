# 研究外部 Skills 模式

## 目标

研究 `mattpocock/skills` 以及 `mindfold-ai/Trellis#326` 中提出的问题和灵感来源，判断哪些思路可以吸收到 DevFlow 项目中。

## 需求

- 下载或以其他方式充分检查相关外部资料，理解其结构、工作流和设计动机。
- 将外部思路与当前 DevFlow 的 skill、task、spec 架构进行对照。
- 输出一份简洁建议，明确区分：
  - 适合立即吸收的想法
  - 需要改造后再吸收的想法
  - 应保持在范围外的想法
- 如果某个吸收路径清晰且风险较低，在修改项目行为前先准备具体实现计划。
- 保留现有 DevFlow 工作流语义，除非研究发现明确更优的替代方案，并且该变更已被显式规划。

## 验收标准

- [ ] 研究结果引用已检查的上游来源，以及用于本地对照的 DevFlow 文件。
- [ ] 输出能用 DevFlow 语境解释 Trellis issue 中的问题和灵感来源。
- [ ] 输出指出可能需要修改的具体 DevFlow 文件、概念或工作流。
- [ ] 推荐变更与观察结论、待决问题分开表达。
- [ ] 在规划产物完成且用户批准从研究/规划进入执行前，不开始实现。

## 备注

- 初始范围是研究和规划。只有当建议足够具体并可供评审后，才进入实现。
